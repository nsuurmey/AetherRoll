This Product Requirements Document (PRD) outlines the core functionality and strategic goals for AetherRoll, a next-generation TTRPG dice utility designed for the 2026 web.1. Product VisionAetherRoll aims to bridge the gap between physical dice satisfaction and digital convenience. By leveraging a "Phygital" aesthetic and decentralized P2P networking, it provides a high-fidelity rolling experience that requires no accounts, no databases, and zero server overhead.2. Target AudienceThe Virtual Player: Needs a fast, synced way to show rolls to a GM without a heavy VTT (Virtual Tabletop).The Mobile Soloist: Wants a "clean yet fun" interface to track complex rolls (like 8d6 fireballs) on the go.The Privacy-Conscious Group: Prefers tools that don't track data or require logins.3. Functional Requirements3.1 Core Rolling EngineStandard Set: Support for d4, d6, d8, d10, d12, d20, and d100.Complex Syntax: Support for modifiers and multi-type rolls (e.g., 1d20 + 2d6 + 4).Special Logic:Exploding Dice: Automatic re-roll on max value if toggled.Advantage/Disadvantage: Single-toggle UI that rolls two d20s and highlights the kept result.3.2 Collaborative Play (P2P)Zero-Backend Sync: Use PeerJS to allow users to host a "Table."Shared Log: All rolls made within a room must appear in the shared history of all connected peers in real-time.Privacy: No roll data is stored on a server; it exists only in the volatile memory of the connected browsers.3.3 User Interface & PresetsQuick Rolls: A "Preset" library allowing users to save and name rolls (e.g., "Sneak Attack").History Log: A sidebar tracking the last 20 rolls with a "clear" button for local sessions.4. Design & Aesthetic (Phygital 2026)FeatureRequirementVisual StyleGlassmorphism 2.0: High-blur backgrounds, 1px "chrome" borders.Color TheoryDeep Teal base with Electric Cyan accents. Crimson flashes for Crits.FeedbackHaptic feedback on mobile; "Liquid" hover states for all buttons.PerformanceMust maintain 60fps animations even on mid-range mobile devices.5. Technical Constraints & Non-Functional RequirementsHosting: Must be fully functional as a static site on GitHub Pages.Lightweight Footprint: The initial JS bundle must be under 500kb to ensure instant loading on mobile data.Reliability: The app must work in "Offline Mode" for solo play using Service Workers (PWA).6. Success MetricsTime to Roll: A first-time user should be able to complete a roll within 3 seconds of page load.Sync Latency: P2P roll broadcasts should appear on peer screens in <200ms.Retention: High usage of the "Presets" feature, indicating the tool has become a staple for the user's campaign.Note on Security: Since this is a P2P application, users are responsible for sharing their Room IDs securely.
